### 2. V2 Bridge (`internal/chat/bridge.go`)
- Connects ArgusCore ‚Üî MessageBus ‚Üî Frontend
- `onCoreMessage(source, content)` ‚Üí forwards to frontend via MessageBus
- `emitStatus()` ‚Üí pushes role status changes (phase:role:status format)
- Status events flow: `Core.emitStatus()` ‚Üí `Bridge.onCoreMessage("status",...)` ‚Üí `app.go onMessage callback` ‚Üí `emitToFrontend("role-status",...)` ‚Üí `frontend EventsOn('role-status')`

### 3. Unified emitToFrontend (`app.go`)
- Single entry point for all backend‚Üífrontend communication
- Auto-generates checksum for each message
- Routes all messages through `MessageBus.Send()` + `EventsEmit()` (dual path for redundancy)
- Writes `[emit‚ÜíFrontend]` log for every event

### 4. Message Tracking Split (`message_bus.go`)
- `shouldTrack()` ‚Äî only `PathCoreOutput` enters pending queue for ACK
- All other paths (system, status, stream, exec, user input) are no-track
- Fixes false "message lost" alarms for system events

### 5. Frontend Timestamp Fix (`frontend/src/App.vue`)
- Changed 5 occurrences of `new Date().toISOString()` ‚Üí `Math.floor(Date.now() / 1000)`
- Root cause: ISO string "2026-05-30T..." ‚Üí `parseInt()` ‚Üí `2026` ‚Üí `new Date(2026000)` ‚Üí "1/1 08:33"
- Now uses Unix seconds, consistent with backend

### 6. Frontend Role-Status Listener (`frontend/src/App.vue`)
- Added `EventsOn('role-status', ...)` handler
- Parses `phase:X|role:Y|status:Z` format
- Updates `aiStatus.pmStatus/seStatus/apStatus` in real-time
- Controls `aiThinking` flag

## In Progress ‚è≥

### 7. MessageBus Context Nil Issue
**Problem:** `NewMessageBus(nil)` in `manager.go:319` ‚Üí `mb.ctx == nil` ‚Üí `Send()` silently returns at line 142 without emitting events.

**Why it worked before:** Dual-path architecture ‚Äî `EventsEmit(a.ctx, payload)` was the primary sender, MessageBus was only for tracking.

**Needed fix:** 
- Add `func (mb *MessageBus) SetContext(ctx context.Context)` method
- Call `msgBus.SetContext(a.ctx)` during app initialization
- Remove `EventsEmit` fallback in `emitToFrontend` ‚Üí single pipe through MessageBus

### 8. emitToFrontend Single-Path
**Goal:** All events go through MessageBus ONLY (single pipe).

**Current state:** Dual path ‚Äî `msgBus.Send()` (tracking) + `EventsEmit()` (actual send). The `EventsEmit` path sends raw payload without `_msgId`, so frontend ACK fails for core_output messages.

**Plan:**
```
Before: msgBus.Send(...) + EventsEmit(...)  ‚Üê 2 pipes
After:  msgBus.Send(...) only               ‚Üê 1 pipe
         ‚Üë with payload merged into data param so frontend gets full content
```

### 9. Hello World Regression
- Last test: SE executed hello.go successfully, output "Hello, World!"
- But message display was broken (frontend received empty/partial data)
- Need full regression after single-path fix
