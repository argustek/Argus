| 列 | 说明 | 控件 |
|----|------|------|
| **#** | 序号（不可编辑） | 纯文本 |
| **URL** | API Base URL | `<input>` 文本框 |
| **模型** | 模型名称 | 文本框（选预制提供商时自动填默认模型） |

## 数据结构变更

### app.go Config（前端 ↔ 后端）

```go
type Config struct {
    APIConfigs      []APIConfig `json:"apiConfigs"`      // 表1：模型配置列表
    PMConfigID      string      `json:"pmConfigId"`       // PM绑定的模型ID（对应apiConfigs[i].id）
    SEConfigID      string      `json:"seConfigId"`       // SE绑定的模型ID
    APEnabled       bool        `json:"apEnabled"`        // AP是否启用
    APConfigID      string      `json:"apConfigId"`       // AP绑定的模型ID（空=共享PM）
    // ... 其他字段不变
}
```

**向后兼容**：旧的 `APConfig *APIConfig` 删除，改为 `APConfigID` 引用方式。
SaveConfig 时根据 ID 查找配置，填充 `types.APIConfig` 传给 Manager。

### SaveConfig 新逻辑

```
1. 遍历 apiConfigs，找默认配置 → 调 UpdateAPIConfig(defaultConfig)
2. 按 pmConfigId 查找 → 调 UpdatePMConfig(pmConfig)
3. 按 seConfigId 查找 → 调 UpdateSEConfig(seConfig)
4. 如果 APEnabled:
     apConfigId 有 → 调 UpdateAPConfig(apConfig)
     apConfigId 空 → 调 UpdateAPConfig(空，回退到aiClient)
   否则 → 调 UpdateAPConfig(空)
```
